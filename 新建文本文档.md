# VitePress 从零搭建到部署完整指南

---

## 一、环境准备

### 1. 安装 Node.js

- 下载地址：https://nodejs.org（推荐 LTS 版本，20.x）
- 安装完成后验证：

```bash
node -v   # 应输出 v20.x.x
npm -v    # 应输出 10.x.x
```

### 2. 安装 Git

- 下载地址：https://git-scm.com
- 安装完成后验证：

```bash
git --version
```

### 3. 配置 Git 用户信息（首次使用必做）

```bash
git config --global user.name "你的名字"
git config --global user.email "你的邮箱@example.com"
```

---

## 二、GitHub 仓库准备

### 1. 创建仓库

1. 登录 GitHub → 右上角 **+** → **New repository**
2. Repository name 填写：`你的用户名.github.io`（例如 `yiyiyiwui.github.io`）
3. 设置为 **Public**
4. **不要勾选** Initialize this repository（本地初始化后推送）
5. 点击 **Create repository**

### 2. 生成 Personal Access Token（PAT）

1. 右上角头像 → **Settings**
2. 左侧最底部 → **Developer settings**
3. **Personal access tokens** → **Tokens (classic)**
4. **Generate new token (classic)**
5. Note 填写备注（如 `vitepress-deploy`）
6. Expiration 选 **No expiration** 或 90 days
7. 勾选 **repo**（第一个大选项，子选项全选）
8. 点击 **Generate token**
9. **立即复制 token，只显示一次！**

### 3. 配置 GitHub Pages（首次部署后操作）

> 第一次推送代码、CI 跑完之后再来设置

1. 进入仓库 → **Settings** → **Pages**
2. Source 选择 **Deploy from a branch**
3. Branch 选择 **gh-pages**，目录选 **/ (root)**
4. 点击 **Save**

## 三、本地创建 VitePress 项目

### 1. 创建项目目录

```bash
mkdir yiyiyiwui.github.io
cd yiyiyiwui.github.io
```

### 2. 初始化 Git 仓库

```bash
git init
git branch -M main
```

### 3. 安装 VitePress

```bash
npm create vitepress@latest
```

> 运行后会有交互提示，配置说明：
> - Where should VitePress initialize the config? → 输入 `docs`（推荐）
> - Site title → 填写你的站点名称（如 `eiの小窝`）
> - Site description → 填写描述
> - Theme → 选 `Default Theme`
> - Use TypeScript → 选 `No`（简单起见）
> - Add VitePress npm scripts → 选 `Yes`

### 4. 安装依赖

```bash
npm install
```

### 5. 目录结构说明

```
yiyiyiwui.github.io/
├── docs/                    # 文档根目录
│   ├── .vitepress/
│   │   └── config.js        # 站点配置文件
│   ├── public/              # 静态资源目录（图片等放这里）
│   │   └── images/
│   │       └── avatar.jpg   # 头像等图片放此处
│   └── index.md             # 首页
├── package.json
└── .gitignore
```

### 6. 创建 .gitignore 文件

在项目根目录创建 `.gitignore`，内容如下：

```
node_modules/
docs/.vitepress/dist/
docs/.vitepress/cache/
```

### 7. 本地预览

```bash
npm run docs:dev
```

浏览器打开 http://localhost:5173 即可预览。

## 四、图片等静态资源使用规范

所有图片放在 `docs/public/` 目录下，**引用时去掉 `/public` 前缀**：

| 文件实际位置 | 配置中引用路径 |
|---|---|
| `docs/public/images/avatar.jpg` | `/images/avatar.jpg` |
| `docs/public/logo.png` | `/logo.png` |

示例（首页 index.md）：

```yaml
hero:
  image:
    src: /images/avatar.jpg   # 正确：不带 /public
    alt: 我的头像
```

## 五、配置 GitHub Actions 自动部署

### 1. 创建工作流文件

在项目根目录创建以下目录和文件：

```
.github/
└── workflows/
    └── deploy.yml
```

### 2. deploy.yml 内容

```yaml
name: Deploy VitePress Site

on:
  push:
    branches: [ main ]

permissions:
  contents: write

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Setup Node.js
        uses: actions/setup-node@v3
        with:
          node-version: 20

      - name: Install dependencies
        run: npm install

      - name: Build VitePress
        run: npx vitepress build docs

      - name: Deploy to GitHub Pages
        uses: peaceiris/actions-gh-pages@v3
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./docs/.vitepress/dist
```

## 六、首次推送到 GitHub

```bash
# 1. 绑定远程仓库（替换为你的用户名和 token）
git remote add origin https://你的用户名:你的token@github.com/你的用户名/你的用户名.github.io.git

# 示例：
git remote add origin https://yiyiyiwui:ghp_xxx@github.com/yiyiyiwui/yiyiyiwui.github.io.git

# 2. 添加所有文件
git add .

# 3. 提交
git commit -m "初始化 VitePress 站点"

# 4. 推送
git push origin main
```

推送后 GitHub Actions 会自动构建并发布到 gh-pages 分支，等待约 1-2 分钟后访问：
**https://你的用户名.github.io**

## 七、日常更新文章/修改内容的完整流程

### 场景一：在当前电脑继续修改

```bash
# 1. 修改/新增文章后，查看改动
git status

# 2. 添加所有改动
git add .

# 3. 提交，-m 后写本次修改说明
git commit -m "新增：xxx 文章"

# 4. 推送，自动触发 CI 重新部署
git push origin main
```

### 场景二：换了电脑，先拉取最新代码

```bash
# 克隆仓库到本地（仅首次在新电脑上执行）
git clone https://你的用户名:你的token@github.com/你的用户名/你的用户名.github.io.git
cd 你的用户名.github.io

# 安装依赖
npm install

# 启动本地预览
npm run docs:dev
```

### 场景三：已经克隆过，再次拉取最新版本

```bash
# 在修改之前先拉取远程最新代码（避免冲突）
git pull origin main

# 然后正常修改、提交、推送
git add .
git commit -m "修改：xxx"
git push origin main
```

## 八、常见问题处理

### 推送被拒绝（rejected）

```bash
# 远程有你本地没有的提交，先拉取再推送
git pull origin main --rebase
git push origin main
```

### Token 失效或 403 错误

重新生成 Token（参考第二章第2步），然后更新远程地址：

```bash
git remote set-url origin https://你的用户名:新token@github.com/你的用户名/你的用户名.github.io.git
```

### 查看当前绑定的远程地址

```bash
git remote -v
```

### 图片不显示

检查引用路径是否带了 `/public`，正确写法是 `/images/xxx.jpg`，不是 `/public/images/xxx.jpg`。

## 十、完整流程总览

```
首次搭建：
创建 GitHub 仓库 → 本地初始化项目 → 配置 CI 文件 → 首次推送 → 配置 Pages

日常更新：
拉取最新(git pull) → 本地修改 → 预览(npm run docs:dev) → 提交推送(git add/commit/push) → 自动部署
```
