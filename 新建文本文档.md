# 📘 VitePress 从零搭建到部署 GitHub Pages 完整指南

> 目标：在一台干净的电脑上，从 0 开始搭建一个 VitePress 博客，并通过 GitHub Actions 自动部署到 `https://你的用户名.github.io`。
>
> 阅读时间：≈ 15 分钟｜动手时间：≈ 30 分钟

## 一、环境准备

### 1. 安装 Node.js

- 下载地址：[https://nodejs.org](https://nodejs.org)（推荐 **LTS 版本，20.x 或更新**）
- 安装完成后打开终端验证：

```bash
node -v   # 应输出 v20.x.x
npm -v    # 应输出 10.x.x
```

> 💡 Windows 安装时一路点 Next 即可。安装完需要**重开**一个终端，否则 `node` 命令找不到。

### 2. 安装 Git

- 下载地址：[https://git-scm.com](https://git-scm.com)
- 安装完验证：

```bash
git --version    # 应输出 git version 2.x.x
```

### 3. 配置 Git 用户信息（首次使用必做）

```bash
git config --global user.name "你的名字"
git config --global user.email "你的邮箱@example.com"
```

这两个信息会写进每次 commit 里。GitHub 上显示的"提交者"就是这两项。

## 二、GitHub 仓库准备

### 1. 创建仓库

1. 登录 GitHub → 右上角 **+** → **New repository**
2. **Repository name** 必须为：`你的用户名.github.io`（例如 `yiyiyiwui.github.io`）
   - ⚠️ 这是 GitHub Pages 的固定规则，用户名仓库才会自动用根域名
3. 设置为 **Public**（私有仓库无法用免费 Pages）
4. **不要勾选** "Initialize this repository with a README"（本地初始化后推送）
5. 点击 **Create repository**

### 2. 生成 Personal Access Token（PAT）

> Token 用来在终端推送代码时代替密码。

1. 右上角头像 → **Settings**
2. 左侧最底部 → **Developer settings**
3. 左侧 **Personal access tokens** → **Tokens (classic)**
4. 右上角 **Generate new token (classic)**
5. **Note** 填备注（如 `vitepress-deploy`）
6. **Expiration** 选 `No expiration` 或 `90 days`
7. 勾选权限：**repo** 整个大项（子项会全选）
8. 点击 **Generate token**
9. ⚠️ **立刻复制 Token**！只显示这一次，关掉就再也看不到了。

把它存到密码管理器或者临时记事本里，下面会用到。

## 三、本地搭建 VitePress 项目

### 1. 创建项目目录

```bash
mkdir yiyiyiwui.github.io
cd yiyiyiwui.github.io
```

### 2. 初始化 Git 仓库

```bash
git init
git branch -M main
```

### 3. 安装 VitePress

```bash
npm create vitepress@latest
```

交互式提示按以下方式回答：

| 提示 | 推荐选择 |
|---|---|
| Where should VitePress initialize the config? | `./docs` |
| Site title | 自定义（如 `我的博客`） |
| Site description | 自定义（如 `德智体美劳全面发展`） |
| Theme | `Default Theme` |
| Use TypeScript for config? | `Yes`（推荐）/ `No` 都可 |
| Add VitePress npm scripts to package.json? | `Yes` |

### 4. 安装依赖

```bash
npm install
```

### 5. 此时的目录结构

```
yiyiyiwui.github.io/
├── docs/
│   ├── .vitepress/
│   │   └── config.mts          # 站点配置（导航、侧边栏、主题）
│   ├── public/                 # 静态资源（图片放这里）
│   │   └── images/
│   └── index.md                # 首页
├── package.json
└── package-lock.json
```

### 6. 创建 `.gitignore`

在项目根目录创建 `.gitignore`：

```
node_modules/
docs/.vitepress/dist/
docs/.vitepress/cache/
.DS_Store
.idea/
```

> 作用：避免把 `node_modules`、构建产物、IDE 配置等上传到 GitHub。

### 7. 本地预览

```bash
npm run docs:dev
```

浏览器打开 [http://localhost:5173](http://localhost:5173)，看到默认主题首页就 ✅。

按 `Ctrl + C` 停止本地服务。

## 四、静态资源（图片）使用规范

所有图片放在 `docs/public/` 目录下，**引用时去掉 `/public` 前缀**：

| 文件实际位置 | 配置中引用路径 |
|---|---|
| `docs/public/images/avatar.jpg` | `/images/avatar.jpg` ✅ |
| `docs/public/logo.png` | `/logo.png` ✅ |
| `docs/public/images/avatar.jpg` | `/public/images/avatar.jpg` ❌ |

首页示例（`docs/index.md`）：

```yaml
---
layout: home
hero:
  name: "我的博客"
  text: 德智体美劳全面发展
  image:
    src: /images/avatar.jpg   # 注意：不带 /public
    alt: 头像
---
```

## 五、配置 GitHub Actions 自动部署

### 1. 创建工作流文件

在项目根目录创建：

```
.github/
└── workflows/
    └── deploy.yml
```

### 2. `deploy.yml` 内容

```yaml
name: Deploy VitePress Site

on:
  push:
    branches: [ main ]

permissions:
  contents: write

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Setup Node.js
        uses: actions/setup-node@v3
        with:
          node-version: 20

      - name: Install dependencies
        run: npm install

      - name: Build VitePress
        run: npx vitepress build docs

      - name: List dist files (debug)
        run: ls -la docs/.vitepress/dist/

      - name: Deploy to GitHub Pages
        uses: peaceiris/actions-gh-pages@v3
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./docs/.vitepress/dist
```

> 这个工作流做了什么：
> 1. 监听 `main` 分支的 push
> 2. 拉代码 → 装 Node.js 20 → `npm install`
> 3. 跑 `vitepress build docs` 生成 `docs/.vitepress/dist/`
> 4. 把 `dist/` 推到 `gh-pages` 分支，由 GitHub Pages 发布

## 六、首次推送到 GitHub

```bash
# 1. 绑定远程仓库（替换为你的用户名和 token）
git remote add origin https://你的用户名:你的token@github.com/你的用户名/你的用户名.github.io.git

# 示例：
git remote add origin https://yiyiyiwui:ghp_xxxxxxxxxxxx@github.com/yiyiyiwui/yiyiyiwui.github.io.git

# 2. 添加所有文件
git add .

# 3. 提交
git commit -m "初始化 VitePress 站点"

# 4. 推送
git push origin main
```

⚠️ Token 写在 URL 里只是为了方便首次推送。如果不想这样，可以用 **SSH 方式**：

```bash
git remote add origin git@github.com:你的用户名/你的用户名.github.io.git
```

然后参考 GitHub 文档配置 SSH key。

推送成功后：

1. 打开仓库主页 → 顶部 **Actions** 选项卡
2. 应该能看到刚刚触发的 `Deploy VitePress Site` 工作流
3. 等待大约 **1~2 分钟**，状态变成 ✅ 即构建完成

## 七、开启 GitHub Pages

> 这一步**第一次 Actions 跑完之后**再来设置。

1. 进入仓库 → 顶部 **Settings**
2. 左侧 **Pages**
3. **Source** 选 `Deploy from a branch`
4. **Branch** 选 `gh-pages`，目录选 `/ (root)`
5. **Save**

等 1 分钟左右，刷新页面，顶部会出现：

```
🎉 Your site is live at https://你的用户名.github.io
```

打开链接就能看到你的博客了。

## 八、日常更新流程

### 场景一：当前电脑继续更新

```bash
# 1. 修改 / 新增 .md 文件
# 2. 本地预览（可选）
npm run docs:dev

# 3. 看一下改了哪些文件
git status

# 4. 添加 → 提交 → 推送
git add .
git commit -m "新增：xxx 文章"
git push origin main
```

推送后 GitHub Actions 自动构建，1~2 分钟后线上生效。

### 场景二：换了电脑，第一次拉代码

```bash
# 克隆仓库（仅首次在新电脑上执行）
git clone https://你的用户名:你的token@github.com/你的用户名/你的用户名.github.io.git
cd 你的用户名.github.io

# 装依赖
npm install

# 启动本地预览
npm run docs:dev
```

### 场景三：在新电脑上已经克隆过，再次开始编辑

```bash
# 编辑之前先拉一下，避免冲突
git pull origin main

# 然后正常修改 → 提交 → 推送
git add .
git commit -m "修改：xxx"
git push origin main
```

## 九、常见问题排查

### ❌ `git push` 被拒绝（rejected）

> 远程有你本地没有的提交。

```bash
git pull origin main --rebase
git push origin main
```

### ❌ Token 失效 / 403 错误

重新生成 Token（第二章第 2 步），更新远程地址：

```bash
git remote set-url origin https://你的用户名:新token@github.com/你的用户名/你的用户名.github.io.git
```

查看当前绑定：

```bash
git remote -v
```

### ❌ 图片不显示

检查路径是否带了 `/public`：
- ✅ 正确：`/images/xxx.jpg`
- ❌ 错误：`/public/images/xxx.jpg`

### ❌ GitHub Actions 跑失败

打开 Actions → 点进失败的那次 → 看红色 ❌ 那一步的日志：

| 报错关键词 | 原因 | 解决 |
|---|---|---|
| `npm ERR! peer dep` | 依赖冲突 | 删本地 `node_modules` 和 `package-lock.json`，重新 `npm install`，再推送 |
| `Permission denied` | 工作流权限不够 | 仓库 Settings → Actions → General → Workflow permissions 选 **Read and write** |
| `dist directory not found` | 构建路径错 | 检查 `publish_dir` 是不是 `./docs/.vitepress/dist` |

### ❌ Pages 显示 404

- 确认 Settings → Pages 中 Branch 选的是 `gh-pages` 而不是 `main`
- 等 2~3 分钟再刷新（首次发布要时间）
- 仓库名必须是 `用户名.github.io`，否则需要在 `config.mts` 加 `base: '/仓库名/'`
